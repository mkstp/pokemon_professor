## Implementation Notes

* **Deterministic damage:** All base damage values are fixed integers.
* **Conditional effects:** Checked at resolution time (e.g., *Correction* reads opponent's last move damage).
* **Single-use modifiers:** Effects apply to the next relevant action only, then expire.
* **Skip-turn logic:** Managed via a boolean flag (`skipNextTurn`).
* **Damage modifiers:** Flat reductions applied after base damage calculation.
* **Preview mechanics:** *Dataset Leak* stores the opponent's next selected move without altering it.
* **Buff/debuff distinction:**
  * *Buff removal* affects the opponent's active modifiers.
  * *Debuff ignore* applies only to the acting entity for that move.

## Design Intent

This move pool supports:

* Predictable but expressive combat
* Readable counterplay (setup → punish loops)
* Light humour grounded in academic stereotypes without breaking mechanical clarity

NPC trainers can be differentiated by:

* Weighted move selection (e.g., aggressive vs. defensive profiles)
* Restricted subsets of this pool
* Scripted sequences for boss encounters
