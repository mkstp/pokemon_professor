# Schema: Deliverables Entry

File: `project_context/deliverables.md`

```
**[DEL-NNN]: [Name]**
[Description — one or two sentences. What this output is.]
Justification: [Why this output is required by the project.]
Issue: [Beads issue ID — created alongside this entry]
```

---

## Notes

- Deliverables are the discrete outputs a stakeholder would receive or evaluate at project completion.
- Each deliverable must have a corresponding Beads issue created at the same time the entry is added. The issue tracks progress; the deliverable entry defines the output.
- Status is not stored here — it is inferred from the state of the linked Beads issue (open = in progress, closed = complete).
- Number sequentially: DEL-001, DEL-002, etc.
