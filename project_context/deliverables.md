# Deliverables

*Entries are added here as deliverables are defined in collaboration with the user. Each entry must have a corresponding Beads issue.*

---

**DEL-001: Playable Browser Game**
A complete, browser-runnable implementation of pokemon_professor. Opens via `index.html` with no build step, no server, and no external dependencies beyond vetted libraries. Contains all seven JS modules implemented per their TDD files. Delivers a fully playable end-to-end core loop: campus navigation, six professor encounters, turn-based battles, dialogue, music, and weather.
Justification: The primary output of the project — the proof-of-concept that validates all design and technical decisions.
Issue: test_project-lkm

---

**DEL-002: Game Design Document**
`project_context/game_design.md` — complete with no placeholders. Covers all required GDD sections: overview, world & setting, player character, all six professors (names, personalities, visual designs, locations, dialogue, moves, HP), campus map, battle system, dialogue system, audio, and weather.
Justification: First-class portfolio artefact documenting the player-facing design before implementation. Demonstrates that design decisions were made deliberately and recorded.
Issue: test_project-1dp

---

**DEL-003: Technical Design Document Suite**
Seven TDD files in `tdd/`: `game.md`, `engine.md`, `data.md`, `map.md`, `battle.md`, `dialogue.md`, `audio.md`. Each defines the module's responsibility, data structures, function contracts, and module interfaces.
Justification: First-class portfolio artefact documenting implementation contracts before code is written. Demonstrates disciplined, spec-first development.
Issue: test_project-bjm *(closed — complete)*

---

**DEL-004: DESIGN.md — Methodology Overview**
A `DESIGN.md` at the repository root explaining the AI-assisted development methodology. Covers: purpose of each artefact (GDD, TDD, session logs, Beads issues), rationale for the flat architecture, the role of AI at each stage, and how a portfolio reviewer can trace decisions from concept to code.
Justification: Portfolio deliverable that makes the development process legible to an external audience. Differentiates this project from a portfolio piece that shows only finished code.
Issue: test_project-xl7
